# Проект: girbo-parser

## Цель
Составить два списка российских компаний:
1. **Майнеры** — компании, которые занимаются добычей криптовалюты
2. **Хостинги** — операторы майнинговой инфраструктуры

## Методология

### Майнеры
Подтверждается 2+ источниками:
- Брали в лизинг майнинговое оборудование (Федресурс)
- ОКВЭД — определяем эмпирически: сначала собираем лизингополучателей, смотрим их ОКВЭД через ФНС, находим паттерн (предварительно замечено: часто 62.09)
- Залог майнингового оборудования (нотариат.ru)
- Реестр майнеров ФНС (ФЗ-221, с 2025)

### Хостинги
- Реестр операторов майнинговой инфраструктуры (ФНС, ФЗ-221) — пока не найден в открытом доступе
- ОКВЭД 63.11 + большие расходы на электроэнергию в бух.отчётности (ГИРБО)
- Федресурс: поиск по "оператор майнинговой инфраструктуры"

## Пайплайн

```
Шаг 1: fedresurs_session.py
       → output/leasing_raw_ДАТА.csv   (лизинг оборудования, СТОП — сохраняем файл)

Шаг 2: fns_enrich.py
       → берёт leasing_raw CSV, по ИНН запрашивает ФНС
       → output/leasing_enriched_ДАТА.csv (+ОКВЭД, название, статус, регион)

Шаг 3: okved_analysis.py
       → топ ОКВЭД среди лизингополучателей → уточняем поисковый паттерн
```

## Текущие файлы
- `fedresurs_session.py` — v7, поиск лизинга на fedresurs.ru
- `.github/workflows/run.yml` — GitHub Actions

## Известные проблемы
- **Qrator блокирует GitHub Actions IP** (ConnectTimeout на всех /backend/* эндпоинтах)
- Решение: запускать **локально на Windows** (домашний IP не заблокирован)
- Для GitHub Actions нужен `PROXY_URL` secret (socks5://user:pass@host:port)

## Ключевые слова для Федресурса
- EXACT (бренды): antminer, whatsminer, bitmain, microbt, canaan, goldshell, jasminer, ebang, ibelink, асик, asic
- BROAD (с доп. фильтром): эвм, вычислительное оборудование, серверное оборудование
