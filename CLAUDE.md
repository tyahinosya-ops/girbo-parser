# girbo-parser — Поиск майнеров и хостингов

## Цель проекта
Составить два верифицированных списка российских компаний:

1. **Майнеры** — компании, добывающие криптовалюту
2. **Хостинги** — операторы майнинговой инфраструктуры (дата-центры для майнинга)

Итоговый результат: два CSV-файла с ИНН, названием, ОКВЭД, источниками подтверждения.

---

## Методология и логика верификации

### Майнер = подтверждён 2+ источниками:
- Брал в лизинг майнинговое оборудование (Федресурс)
- ОКВЭД совпадает с паттерном (определяем эмпирически, см. ниже)
- Залог майнингового оборудования (нотариат.ru)
- Числится в реестре майнеров ФНС (ФЗ-221, обязательная регистрация с 2025)

### Хостинг = подтверждён 2+ источниками:
- В реестре операторов майнинговой инфраструктуры (ФНС, ФЗ-221)
- Большие расходы на электроэнергию в бух.отчётности (ГИРБО/ФНС)
- ОКВЭД 63.11 (обработка данных, хостинг)
- Упоминание на Федресурсе по запросу "оператор майнинговой инфраструктуры"

### Ключевой инсайт по ОКВЭД:
> **Не угадывать ОКВЭД заранее.** Сначала собираем лизингополучателей через
> Федресурс → по их ИНН смотрим реальные ОКВЭД в ФНС → находим паттерн.
> Пользователь заметил, что большинство майнеров используют **ОКВЭД 62.09**
> (деятельность в области IT), а не 64.19.x или 26.20, как можно было ожидать.

## Пирамида источников данных

```
Уровень 1 — BULK без блокировок (приоритет)
  data.nalog.ru          ФНС ЕГРЮЛ bulk XML   → ОКВЭД, статус, регион компании
  egrul.nalog.ru         ФНС по ИНН (JSON)    → обогащение по конкретным ИНН
  nalog.gov.ru           Реестр майнеров ФНС  → пока не найден в open data

Уровень 2 — API с лёгкими ограничениями
  notariat.ru            Реестр залогов        → АСИК в залоге
  fssp.gov.ru            Долги ФССП            → скоринг надёжности
  kad.arbitr.ru          Арбитраж              → споры с лизингодателями

Уровень 3 — Scraping (Qrator → нужен прокси)
  fedresurs.ru           Лизинговые сообщения  → ГЛАВНЫЙ источник
```

```
Шаг 1  fedresurs_session.py
       ↓ ключевые слова (бренды + общие термины)
       → output/leasing_raw_ДАТА.csv
         Колонки: keyword, lessee_inn, lessee_name, lessor_inn,
                  lessor_name, date, subject, msg_id, msg_url
       ← СТОП. Файл сохраняется. Следующий шаг запускается отдельно.

Шаг 2  fns_enrich.py  [НЕ НАПИСАН]
       ↓ берёт leasing_raw CSV, читает lessee_inn
       → запрашивает ФНС (egrul.nalog.ru или data.nalog.ru)
       → output/leasing_enriched_ДАТА.csv
         + колонки: okved_main, okved_list, company_status,
                    reg_date, region, authorized_capital

Шаг 3  okved_analysis.py  [НЕ НАПИСАН]
       ↓ берёт leasing_enriched CSV
       → топ ОКВЭД среди лизингополучателей
       → вывод: какой ОКВЭД искать для расширения списка майнеров
```

## fedresurs_session.py — детали реализации

### Ключевые слова (два типа):

**EXACT** — бренды, однозначно майнинг, берём без доп. фильтра:
```
antminer, bitmain, whatsminer, microbt, canaan, avalon miner,
innosilicon, goldshell, jasminer, ebang, ibelink,
асик, asic, майнинговое оборудование, оборудование для майнинга,
оборудование для добычи криптовалют, добыча криптовалюты,
майнинг криптовалют, хэшрейт, hashrate
```

**BROAD** — общие термины, принимаем только если в тексте есть MINING_INDICATORS:
```
эвм, вычислительное оборудование, серверное оборудование
```

**MINING_INDICATORS** (для фильтрации BROAD):
```
antminer, whatsminer, bitmain, microbt, canaan, innosilicon,
goldshell, jasminer, ebang, ibelink, асик, asic, майнинг,
mining, криптовалют, хэшрейт, hashrate, blockchain, блокчейн,
добыча цифровой, цифровая валюта
```

### Эндпоинт-дискавери:
Скрипт перебирает 10 вариантов эндпоинтов (GET/POST × разные пути),
ищет первый рабочий. fedresurs.ru менял API без предупреждений.

### Прокси:
Читает из env: `PROXY_URL` → `HTTPS_PROXY` → `ALL_PROXY`
Формат: `socks5://user:pass@host:port` или `http://host:port`
В логах скрывает user:pass (показывает только host:port).

### Дедупликация:
По `msg_id`, или если нет — по `lessee_inn|date|subject[:40]`

### Сортировка вывода:
CSV сортируется по `date` DESC (новые записи сверху)

## Известная проблема: Qrator

**Симптомы:**
- `GET fedresurs.ru` → HTTP 401, кука `qrator_jsr` (JS-challenge, httpx не может решить)
- Все `/backend/*` → `ConnectTimeout` (TCP-дроп на уровне сети)

**Причина:** fedresurs.ru стоит за Qrator (российский DDoS-фильтр).
Qrator блокирует IP-диапазоны Azure (GitHub Actions) на уровне TCP.

**Решения (по простоте):**
1. ✅ Запустить локально на Windows — домашний IP не заблокирован
2. 💰 Купить SOCKS5 прокси (~200 руб/мес, proxyline.ru) → secret `PROXY_URL`
3. 💰 Self-hosted GitHub Actions runner на VPS с нормальным IP
