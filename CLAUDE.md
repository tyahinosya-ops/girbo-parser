# girbo-parser — Поиск майнеров и хостингов

## Цель проекта
Составить два верифицированных списка российских компаний:

1. **Майнеры** — компании, добывающие криптовалюту
2. **Хостинги** — операторы майнинговой инфраструктуры (дата-центры для майнинга)

Итоговый результат: CSV-файл с ИНН, названием, ОКВЭД, скорингом и источниками подтверждения.

---

## Методология

### Путь А — Майнеры (лизингополучатели)
1. Федресурс → поиск по ключевым словам (antminer, whatsminer, bitmain и др.)
2. Извлекаем ИНН **лизингополучателей** (те, кто брал оборудование)
3. По ИНН получаем все ОКВЭДы из ЕГРЮЛ → анализируем паттерны
4. Проверяем залоги на notariat.ru (ASIC в залоге = сильный сигнал)
5. Проверяем реестр майнеров ФНС (Закон №259-ФЗ)
6. Из ГИРБО берём финансы: ОС (оборудование дорогое), выручка/сотрудник

### Путь Б — Хостинги (лизингодатели)
1. Те же договоры на Федресурсе → извлекаем ИНН **лизингодателей**
2. Фильтруем финансовые организации (банки, ОКВЭД 64.x) — они посредники
3. Остальные лизингодатели → потенциальные хостинги
4. Проверяем ГИРБО: большие расходы на электроэнергию = ЦОД
5. Ищем ОКВЭД 63.11 (обработка данных, хостинг)

### Перекрёст А ∩ Б → максимальный приоритет

### Ключевой инсайт по ОКВЭД:
> Не угадывать ОКВЭД заранее. Сначала собираем лизингополучателей →
> смотрим их реальные ОКВЭДы → находим паттерн эмпирически.
> Майнеры часто используют **62.09** (прочая ИТ-деятельность).

## Архитектура пайплайна

```
python main.py --mode fedresurs
       │
       ▼
pipeline/fedresurs.py — search_by_keywords()
       │                  ключевые слова: antminer, whatsminer, bitmain,
       │                  microbt, асик, asic, майнер, эвм, криптовалют
       ├── miner_inns    (лизингополучатели) ──→ ПУТЬ А
       └── hosting_inns  (лизингодатели)    ──→ ПУТЬ Б
                │
                ▼
pipeline/enrichment.py — enrich_batch()
       ├── pipeline/rusprofile.py   (Playwright — финансы, статус)
       ├── pipeline/fns.py          (ГИРБО + ЕГРЮЛ + реестры ФНС)
       └── pipeline/notariat.py     (залоги ASIC на notariat.ru)
                │
                ▼
pipeline/filters.py — apply_cascade_filters()
       │   выручка > 1млн | ОС > 500к | ЭЭ > 500к (если есть)
       │
       ▼
pipeline/scoring.py — score_all()
       │   Базовые веса: лизинг(40%) + ОС(25%) + ЭЭ(20%) + выручка(15%)
       │   Бонусы: регион(+15) + год_регистрации(+12) +
       │           тип_лизингодателя(+20) + перекрёст_путей(+15) +
       │           ASIC_залог(скоринг учитывает через leasing_score)
       │
       ▼
output/leads_fedresurs_ДАТА.csv
```

## Скоринг (0–100 баллов)

### Базовые веса
| Сигнал | Вес |
|--------|-----|
| Лизинговые ключевые слова | 40% |
| Основные средства (ОС) | 25% |
| Расходы на электроэнергию | 20% |
| Выручка | 15% |

### Бонусы (приоритизация, не жёсткий отсев)
| Бонус | Баллы |
|-------|-------|
| Регион с дешёвой ЭЭ (Иркутск, Хакасия) | до +15 |
| Год регистрации в крипто-волну (2017/2021) | до +12 |
| Лизингодатель — ЦОД (ОКВЭД 63.x) | +20 |
| Перекрёст путей А и Б | +15 |
| 0 сотрудников + выручка | +8 |

### Приоритеты
- 🔥 **Горячий** — ≥70 баллов
- 🌡 **Тёплый**  — 40–69 баллов
- ❄️ **Холодный** — <40 баллов

## Запуск

```bash
# Установка зависимостей
pip install -r requirements.txt
playwright install chromium

# Основной режим (Федресурс → два пути)
python main.py --mode fedresurs

# Без записи в БД
python main.py --mode fedresurs --skip-db

# Режим ЕГРЮЛ (старый)
python main.py --mode egrul --category mining

# Анализ ОКВЭДов по результату
python analyze_okved.py output/leads_fedresurs_2026-03-19.csv --export okved_report.csv

# Только горячие
python analyze_okved.py output/leads_fedresurs_2026-03-19.csv --priority Горячий
```

## Известная проблема: Qrator (Федресурс)

**Симптомы:** HTTP 401 или ConnectTimeout на fedresurs.ru из облачных IP (GitHub Actions, Azure).

**Решения:**
1. ✅ Запустить локально — домашний IP не заблокирован
2. 💰 SOCKS5 прокси (~200 руб/мес) → переменная `PROXIES` в config.py
3. 💰 Self-hosted GitHub Actions runner на VPS

## Открытые вопросы

1. **Реестр ФНС (Закон №259-ФЗ)** — публичный API не найден.
   `FNS_MINER_REGISTRY_URL` в fns.py — заглушка, при 404 возвращает False.
2. **notariat.ru** — структура API/HTML может меняться, нужно проверить на реальных данных.
3. **После первого прогона** — запустить `analyze_okved.py` и обновить
   `TARGET_OKVEDS` в config.py на основе реальных паттернов.
